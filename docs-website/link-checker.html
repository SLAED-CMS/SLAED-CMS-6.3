Link Checker

This tool checks all documentation links for broken references.

How it works: This tool sends HTTP HEAD requests to each documentation page to verify if the links are working. A status code in the 200-399 range indicates a working link.

Note: For more reliable results, try the Enhanced Link Checker which uses server-side PHP.

Troubleshooting

If links show as broken:

  • Verify the file exists in the documentation directory
  • Check that the filename matches exactly (case-sensitive)
  • Ensure there are no typos in the filename
  • If running locally, some browsers may block these requests due to security restrictions

Note: When running this tool locally, some browsers may show false positives due to CORS restrictions. For accurate results, run this tool on a web server.
