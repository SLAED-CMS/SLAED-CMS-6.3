Link-Prüfer

Dieses Werkzeug prüft alle Dokumentationslinks auf fehlerhafte Verweise.

So funktioniert es: Dieses Werkzeug sendet HTTP HEAD-Anfragen an jede Dokumentationsseite, um zu überprüfen, ob die Links funktionieren. Ein Statuscode im Bereich 200-399 zeigt einen funktionierenden Link an.

Hinweis: Für zuverlässigere Ergebnisse verwenden Sie den Erweiterten Link-Prüfer, der serverseitiges PHP verwendet.

Fehlerbehebung

Wenn Links als fehlerhaft angezeigt werden:

  • Überprüfen Sie, ob die Datei im Dokumentationsverzeichnis existiert
  • Stellen Sie sicher, dass der Dateiname genau übereinstimmt (Groß-/Kleinschreibung beachten)
  • Vergewissern Sie sich, dass keine Tippfehler im Dateinamen vorhanden sind
  • Bei lokaler Ausführung können einige Browser diese Anfragen aufgrund von Sicherheitseinschränkungen blockieren

Hinweis: Bei lokaler Ausführung dieses Werkzeugs können einige Browser aufgrund von CORS-Einschränkungen falsche Ergebnisse anzeigen. Für genaue Ergebnisse führen Sie dieses Werkzeug auf einem Webserver aus.
